🧠 Si trabajas con IA para generar contenido, landing pages o assets, Git no es opcional.

No porque sea "técnico". S porque te va a salvar la vida el día que algo se rompa.

Te explico los conceptos clave:

**¿Qué es Git?**
Control de versiones. Cada vez que guardas un cambio, Git crea un snapshot de tu proyecto en ese momento. Como un historial de Google Docs, pero local y más potente.

**Working tree**
Tu carpeta de archivos tal como la ves. Aquí editas, creas, borras. Git observa los cambios que haces aquí y te dice qué ha cambiado.

**Commits**
Un checkpoint. "Guardé esto así, a las 3pm, con este mensaje." Cada commit tiene un hash único que lo identifica para siempre.

**Los comandos que necesitas saber:**

1️⃣ `git init` — Crea un repositorio nuevo
2️⃣ `git commit` — Guarda un snapshot
3️⃣ `git push` — Sube a la nube
4️⃣ `git pull` — Baja cambios de la nube
5️⃣ `git checkout` — Salta entre versiones o ramas
6️⃣ `git rebase` — Reordenar tu historia
7️⃣ `git reset` — Deshacer o volver atrás

**Buenos commit messages:**

Formato: `<tipo>(<scope>): <descripción>`

✅ `feat(campaign): add holiday page`
✅ `fix(tracking): correct GA4 events`
❌ `update`
❌ `fix`
