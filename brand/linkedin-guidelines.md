## Carousels (Documentos PDF)

### Estructura estandar
- **Slide 1**: Hook + titulo + author-bar (avatar + nombre + puesto)
- **Slides 2-N**: Contenido principal (1 idea por slide)
- **Slide final**: CTA + author-bar + monograma

### Reglas de composicion
- Maximo 10-12 slides (LinkedIn limita a 100 paginas PDF pero menos es mejor)
- Cada slide es independiente: debe funcionar solo
- Titulo grande y legible en mobile
- Mucho espacio en blanco, no llenar todo el slide
- Numerar slides (excepto el primero y ultimo)
- Author-bar (avatar + "Alex Spelucin" + puesto: "SEO Técnico para Shopify" ES / "Shopify Technical SEO" EN) en **todos** los slides
- Monograma: SOLO en el slide final, nunca repetido junto a la author-bar en otros slides

### Tipografia en carousel
- Titulo: 56-64px, weight 700
- Subtitulo: 32-36px, weight 500
- Body: 24-28px, weight 400
- Caption: 16-18px, weight 500, uppercase, tracking 0.18em
- Metricas: 72-96px, weight 300

### Colores
- Fondo: `#f4fdff` o `#eaf6ff`
- Texto principal: `#1560bd`
- Acentos: `#0d4a9e`
- Dot grid sutil de fondo

### Anatomia de slide
```
┌─────────────────────────────────┐
│  [eyebrow: categoria]           │
│                                 │
│  TITULO GRANDE                  │
│  subtitulo o descripcion        │
│                                 │
│  ┌─────────────────────────┐   │
│  │  contenido visual       │   │
│  │  (diagrama, metrica,    │   │
│  │   lista, codigo)        │   │

```

## Banners (Imagenes de post)

### Tipos
1. **Announcement**: Lanzamiento de herramienta, nuevo servicio
2. **Quote**: Frase provocativa o insight
3. **Data**: Metrica o estadistica visual
4. **Tutorial**: Paso a paso visual
5. **Comparison**: Antes/despues, A vs B

### Reglas
- Headline legible en mobile (texto grande)
- Maximo 2-3 lineas de texto
- CTA visual si aplica
- Dot grid sutil de fondo
- Gradiente diagonal suave

### Anatomia de banner (1200x667)
```

│                                                         │

│                                                         │
│  Subtitulo o CTA                                        │

│                                          [monograma]    │

```

## Cover de artículos (744x400)

### Reglas
- Solo texto: título + eyebrow (categoría) + monograma
- NO incluir avatar/foto ni CTA
- Headline legible en mobile
- Maximo 3 lineas de título
- Dot grid sutil de fondo
- Mucho espacio en blanco

## Posts de texto (sin imagen)

### Estructura de copy
Ver `knowledge/voice-guide.md` para la voz completa.

### Formato
- Primer comentario: links, CTAs adicionales
- Hashtags: maximo 1-2, al final o en primer comentario
- Bio signature: solo en thought leadership posts
- Divider "--" antes de la signature

---

## Frecuencia y calendario

### Sugerencia
- **3-4 posts por semana** (calidad sobre cantidad)
- **1 carousel por semana** (contenido profundo)
- **2-3 posts de texto** (insights, opiniones, tips)
- **1 banner** cuando haya announcement o launch
- **1-2 artículos por mes** (contenido profundo, evergreen)

### Mejores horarios (LATAM)
- Martes a jueves: 8-10am o 12-1pm
- Evitar fines de semana para contenido tecnico

## Engagement

### CTAs efectivos
- "Comenta X para..." (lead magnet)
- "Link en primer comentario" (para links externos)
- "Charlamos?" (soft CTA)
- "Cual es tu experiencia con...?" (engagement)

### Evitar
- Pedir likes o shares explicitamente
- "Etiqueta a alguien que..."
- Engagement bait sin sustancia

## Artículos LinkedIn

### Qué son
Artículos nativos de LinkedIn (no posts largos). Contenido largo con portada, publicado directamente en LinkedIn.

### Ventajas
- Mayor alcance orgánico que posts normales
- SEO en buscadores de LinkedIn y Google
- Posicionamiento como thought leadership
- Contenido evergreen en el feed

### Estructura de artículo
- **Título**: 5-10 palabras, claro y específico
- **Subtítulo**: Contexto adicional (opcional)
- **3-5 secciones** con H2 (`##`)
- **Longitud**: 1000-2000 palabras (ideal: 1200-1500)
- **Conclusión** con resumen + CTA suave

### Formato markdown para LinkedIn
- `##` para secciones principales (H2)
- `###` para subsecciones (H3)
- `**bold**` para enfasis
- `- ` para listas con bullet points
- `1. ` para listas numeradas
- `> ` para blockquotes (insights, frases clave)
- `---` para separadores visuales
- No usar H1 (`#`) — LinkedIn lo reserva para el título

### Reglas de escritura
- Párrafos cortos (máximo 3-4 oraciones)
- Code-switching natural (términos técnicos en inglés)
- Blockquotes para insights que necesitan destacar
- Ejemplos concretos en cada sección
- Code fences para configuraciones, queries, scripts
